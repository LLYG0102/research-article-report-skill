# 子图解读模板

```markdown
Fig. {number}{letter} 展示的是{实验/模型/流程/结构/统计结果}。图中的{横轴/纵轴/颜色/结构标注/曲线/显微图读出}分别表示{具体含义}。这个子图支持的结论是{结论}，依据是{关键数值、对照、趋势或结构一致性}。它在全文证据链中的作用是把{上一层证据}推进到{下一层证据}。
```

## 不同图型的关注点

- 流程图：说明输入、输出、中间模块、决策阈值和与旧方法的差异。
- 模型结构图：说明表示、信息流、训练目标、约束或 conditioning。
- benchmark 图：说明任务定义、数据划分、baseline、指标和统计显著性。
- 消融图：说明去掉哪个模块、性能如何变化、证明哪个设计选择必要。
- 结构图：区分设计模型、预测结构和实验结构，报告 RMSD/resolution/pAE 等指标。
- 显微/电镜图：说明样品、尺度、视角、class average/reconstruction 与设计模型如何对应。
- 结合/功能曲线：说明浓度、读出、Kd/IC50/EC50/Emax、阳性/阴性对照，以及 binding 是否等同于 function。

## 示例

```markdown
Fig. 2b 是 calcium mobilization assay，用来判断候选 binder 是否真正激活受体。横轴是配体浓度，纵轴是钙信号响应，曲线形状反映剂量依赖性。若某设计蛋白达到接近内源配体的 Emax 并有合理 EC50，这个子图就把“筛到结合蛋白”推进为“具有 GPCR agonist 功能”。如果只有结合信号而没有功能读出，则不能写成 agonist。
```
